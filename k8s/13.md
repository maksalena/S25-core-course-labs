# Kubernetes ArgoCD Deployment

## Setting Up ArgoCD

I installed ArgoCD and verified the installation:

```bash
kubectl wait --for=condition=ready pod -l app.kubernetes.io/name=argocd-server -n argocd --timeout=90s
pod/argo-argocd-server-6f848cf7f7-5p254 condition met
```

Then set the port forwarding:

```bash
kubectl port-forward svc/argo-argocd-server -n argocd 8080:443
Forwarding from 127.0.0.1:8080 -> 8080
Forwarding from [::1]:8080 -> 8080
Handling connection for 8080
Handling connection for 8080
...
```

Then I logged in and created the ArgoCD application configuration:

```bash
kubectl apply -f k8s/ArgoCD/argocd-python-app.yaml
application.argoproj.io/python-app configured
```

## ArgoCD Behavior Explanation

ArgoCD is a declarative, GitOps-based continuous delivery tool for Kubernetes applications. It continuously monitors the actual state of Kubernetes resources and ensures they match the desired state as defined in a Git repository.

* Configuration Drift: Whenever manual changes are made to a Kubernetes resource outside of the Git-defined configuration, ArgoCD detects this drift and marks the application as OutOfSync. It can then either prompt a manual sync or automatically restore the correct configuration based on the repository settings.

* AutoSync: If enabled, ArgoCD will automatically reconcile any differences between the cluster state and the Git repository, ensuring that deployments stay aligned with the intended configurations. This helps maintain stability and consistency across environments without manual intervention.

* Pod Recovery: When a pod is manually deleted or modified, Kubernetes itself attempts to restore it based on the deployment specifications. However, if any changes affect the overall deployment configuration (such as replica count), ArgoCD detects this and ensures that the specified number of replicas and configurations are reinstated according to the Git repository.
